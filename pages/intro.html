Chapter #1: Get Started

#1: Programs and Programming Languages

Lesson Module: Programs and Programming Languages

Learning Objectives

After completing this lesson, you should be able to:

1. Define what a program and programming language are.

2. Understand the concept of machine language and its binary nature.

3. Explain the purpose of assembly language and the role of an assembler.

4. Describe high-level programming languages and their advantages.

5. Differentiate between compilers and interpreters in translating source code to machine code.

Lesson Content

1. Introduction to Programs and Programming Languages

• A program (or software) is a set of instructions that tell a computer what to do.

• Programs are used to interact and communicate with computers.

• To write programs, we use programming languages.

• Programming languages are languages that are used to write programs and are understood by computers.

2. Machine Language

• Machine language is the computer's native language, using only 0s and 1s (binary).

• It is the main language of the computer, and all instructions must eventually be translated into machine language before execution.

• Machine language is machine-dependent, meaning it differs among different types of machines.

• Example: This is how you add 2 and 3 (2+3) in machine language: 1010101010101010

3. Assembly Language

• Assembly language was developed to make programming easier compared to machine language.

• It is also machine-dependent, but it uses keywords (e.g., add, sub) instead of binary codes.

• Assembly code must be translated to machine code using a program called an assembler before execution.

• Example: This is how you add 2 and 3 in assembly language: add 2, 3, result

4. High-Level Programming Languages

• High-level programming languages use English words and are easier to learn and use.

• They are machine-independent, meaning programs can run on different machines.

• Each instruction in a high-level language is called a statement.

• A program written in a high-level language is called source code or a source program.

• Example: This is how you add 2 and 3 in Java (a high-level language): result = 2 + 3;

5. Translating Source Code to Machine Code

• Source code written in high-level languages cannot be directly executed by computers.

• It must be translated into machine code before execution.

• This translation can be done using a compiler or an interpreter.

• A compiler translates the entire source code into machine code at once, producing an executable file.

• An interpreter translates and executes each statement of the source code one by one.

Review Questions

1. What is a program, and what is its purpose?

2. Explain the difference between machine language, assembly language.

3. What are the advantages of using high-level programming languages?

4. Describe the roles of a compiler and an interpreter in translating source code to machine code.

#2: Introduction to Java Programming

Lesson Module: Introduction to Java Programming

Learning Objectives

After completing this lesson, you should be able to:

1. Understand the general information about Java and its development.

2. Explain the Java Language Specification and its syntax and semantics.

3. Define what an API (Application Programming Interface) is and its purpose.

4. Describe the different editions of Java and their use cases.

5. Understand the components of the Java Development Kit (JDK) and their roles.

6. Identify what an Integrated Development Environment (IDE) is and its features.

Lesson Content

1. General Information about Java

• Java was developed at Sun Microsystems (acquired by Oracle in 2010).

• It is a general-purpose and powerful programming language.

• Java is used for developing software that runs on mobile, desktop, and server platforms.

• Java is machine-independent, meaning programs can run on different machines.

2. Java Language Specification

• The Java Language Specification defines the syntax and semantics of Java.

• Syntax refers to the grammar rules for writing valid Java code.

• Semantics refers to the meaning and behavior of Java code.

• Syntax errors occur when code violates the grammar rules.

• Semantic errors occur when code is grammatically correct but lacks meaning.

3. Application Programming Interfaces (APIs)

• An API, or library, contains predefined Java code that developers can use.

• APIs provide ready-to-use code, making development faster and easier.

• Developers don't have to write everything from scratch.

4. High-Level Programming Languages

• Java Standard Edition (Java SE): Used for developing desktop applications.

• Java Enterprise Edition (Java EE): Used for developing server-side applications.

• Java Micro Edition (Java ME): Used for developing mobile applications.

• Java SE is the foundation for all other editions.

5. Java Development Kit (JDK)

• The JDK is a set of programs that enable Java development.

• It includes the Java Runtime Environment (JRE) for running Java programs.

• Both JDK and JRE contain the Java Virtual Machine (JVM).

• The JVM executes Java programs on different machines, making Java machine-independent.

6. Integrated Development Environment (IDE)

• An IDE is a program that provides an environment for writing, compiling, building, and running Java programs.

• IDEs typically include tools for debugging and finding errors.

• Popular Java IDEs include NetBeans, Eclipse, and IntelliJ IDEA

Review Questions

1. What is the purpose of the Java Language Specification?

2. Explain the difference between syntax and semantics errors in Java.

3. Why are APIs (libraries) useful in Java development?

4. What are the different editions of Java, and what are their use cases?

5. Describe the roles of JDK, JRE, and JVM in Java development.

6. What are the main features and benefits of using an Integrated Development Environment (IDE)?
